Splash screen — the arrival moment (Mobile)

Today's splash on the left, the proposal on the right — both clocked. A rebuild of components/SplashScreen. The shipped component holds the guest for load + 5.0 s, closes two curtains over its own logo at 4.3 s, and — if the auth request hangs — never leaves. Here the splash is a handoff, not a gate: it covers the skeleton of the screen it is about to become, leaves the instant the data lands, and when the data never lands it says so and offers a way out.

Two treatments, one behaviour. The Style switch picks between Classic — the refined V1: flat brand background, bare logo, three pulsing dots, curtains removed — and Modern — the pre-auth idiom already used by Find booking and Sign-in: branded gradient, colour orbs, film grain, a liquid-glass plate and the honest progress ring. Every fix ships in both. Mock data only.

Curtains closed at 4.3 s. For the next 0.0s the guest is looking at a solid rectangle of the brand colour — no logo, no loader, no text, no way out.

simulated network · the 5 s timer starts after the data arrives

The 5 s tax. SHOW_MODAL_AFTER_SPLASH_TIMEOUT = 5000 — and the timer only starts once hideSplashScreen flips true. Total splash time is load + 5000 ms on every cold start, forever, for everyone.

The infinite splash. That timer is the component's only timer, and it is gated behind the data having already arrived. If the auth request hangs — offline, captive portal, dead API — hideSplashScreen never flips, and the splash never unmounts. There is no watchdog.

The late splash. isSplashScreenEnabled reads splash_screen_enabled from branding, which arrives after first paint. So the app renders, and then the splash slams down on top of it. The logo and the brand colours land late too, on top of the default ones.

Dead CSS. animation-timing-function: cubic-bezier(0, 1.23) takes two arguments where CSS requires four, so the declaration is dropped and the curtains silently fall back to ease. The intended overshoot has never once rendered.

The timing contract

500 ms minimum — a splash that flashes for 90 ms is worse than none.
1.2 s before any text — narrating a fast load is noise.
6 s → admit it. Elapsed clock, Try again, and a promise that nothing is lost.
20 s → watchdog. Stop pretending; render the error state with real exits.
Today: load + 5000 ms, with no upper bound at all.
